# MIT 딥러닝 입문 - Day 2: Computer Vision and CNNs

---

## 주제
- 컴퓨터에게 인간의 시각(vision) 능력을 부여하는 방법에 대한 강의
- 이미지 데이터로부터 의미 있는 정보를 추출하고 학습하는 딥러닝 기법 소개
- Convolutional Neural Networks(CNN)의 원리와 다양한 응용 분야 탐구
- 단순 분류(classification)부터 객체 감지(object detection), 세그멘테이션(segmentation), 자율주행까지 컴퓨터 비전의 폭넓은 응용 사례 학습

---

## 핵심개념 정리

### 1. Vision(시각)의 개념과 중요성

**[1-1] Vision의 정의**
- "What is where by looking" - 보는 것만으로 무엇이 어디에 있는지 아는 능력
- 인간의 가장 중요한 감각 중 하나로, 감정 해석, 세상 탐색, 물체 조작 등을 가능하게 함

**[1-2] Vision의 특성**
- 정적 이해뿐만 아니라 동적 이해도 포함
  - 예: 도로 이미지에서 주차된 차와 움직이는 차를 구분
  - 사람들의 정지 상태와 이동 상태를 인식
- 장면의 모든 세부사항을 종합적으로 해석하는 능력

**[1-3] Computer Vision의 응용 분야**
- 로봇공학 및 조작(robotics, manipulation)
- 자율주행(autonomous driving)
- 모바일 컴퓨팅(스마트폰의 임베디드 하드웨어)
- 생물학, 의료, 헬스케어(의료 영상 진단)
- 접근성 애플리케이션(accessibility applications)

---

### 2. 이미지의 표현 방식

**[2-1] 컴퓨터의 이미지 인식**
- 이미지는 컴퓨터에게 숫자의 배열에 불과함
- 언어보다 처리가 쉬운 이유: 이미지는 이미 2차원 숫자 배열로 표현되어 있음

**[2-2] 이미지 표현 구조**
- 그레이스케일 이미지: 2차원 배열, 각 픽셀은 하나의 숫자(밝기 값)
- 컬러 이미지: 3차원 배열 (2차원 공간 + 1차원 색상 채널)
  - 색상 채널은 항상 길이 3 (Red, Green, Blue 값의 조합)

---

### 3. 머신러닝의 주요 작업 유형

**[3-1] Regression(회귀)**
- 연속적인 값을 출력하는 작업
- 예: 자율주행에서 조향각 예측

**[3-2] Classification(분류)**
- 클래스 레이블을 출력하는 작업
- n개 또는 k개의 이산적(discrete) 클래스 중 하나를 선택
- 예: 이미지 속 미국 대통령 식별

---

### 4. Feature Detection(특징 검출)

**[4-1] Feature Detection의 개념**
- 각 클래스를 구성하는 고유한 요소가 무엇인지 이해하고 정의하는 작업
- 한 클래스를 다른 클래스와 구별하는 특징을 추출하는 과정

**[4-2] 분류의 2단계 프로세스**
1. 특징 정의: 이미지에서 무엇을 찾을지 정의
2. 특징 검출: 정의된 특징들을 실제로 찾아냄
- 충분한 특징을 검출하면 해당 객체로 분류 가능

**[4-3] Feature Detection의 예시**
- 얼굴 감지: 눈, 코, 귀를 찾음
- 집 감지: 문, 창문, 계단을 찾음

**[4-4] Computer Vision의 어려움**
- 동일한 객체도 다양한 변형으로 나타남
  - 시점(viewpoint) 변화
  - 스케일(scale) 변화
  - 가려짐(occlusion)
  - 조명(illumination) 변화
- 같은 물체를 다른 카메라로 찍어도 픽셀 공간에서 매우 다르게 나타남
- 분류 알고리즘은 이러한 모든 변화에 불변(invariant)해야 함

---

### 5. Machine Learning vs Deep Learning

**[5-1] Machine Learning의 한계**
- 특징을 사람이 수동으로 정의해야 함
- 재귀적 문제 발생: 얼굴을 눈으로 감지하려면, 눈을 어떻게 감지할지도 정의해야 함

**[5-2] Deep Learning의 접근법**
- 데이터를 사용하여 특징을 학습
- 사람이 특징을 정의하지 않고, 많은 이미지를 보여주면 모델이 자동으로 고유한 특징 추출
- 계층적 방식으로 특징 학습: 네트워크의 깊이(depth)가 계층 구조를 생성

**[5-3] Neural Network의 계층적 학습**
- 각 레이어가 이전 레이어의 출력으로부터 구성됨
- 레이어가 깊어질수록 더 복잡하고 표현력 있는 특징 학습 가능
- 명시적 정의 대신 암묵적 학습을 통한 계층 구조 형성

---

### 6. Fully Connected Networks의 한계

**[6-1] 이미지 처리의 문제점**
- 2차원 이미지를 1차원으로 평탄화(flatten)해야 함
- Lecture 1에서 배운 fully connected network는 1차원 입력만 처리 가능

**[6-2] 평탄화의 문제**
- 공간 정보가 100% 손실됨
- 이미지의 풍부한 공간 구조 정보를 완전히 파괴
- 모든 픽셀이 모든 뉴런에 연결되어 매개변수 수가 매우 많음
- 계산 비용이 매우 높고 차선의 솔루션 제공

**[6-3] 해결 과제**
- 데이터에 내재된 풍부하고 고유한 공간 정보를 보존하고 활용하는 방법 필요
- 네트워크 아키텍처에 이 정보를 반영해야 함

---

### 7. Convolutional Neural Networks(CNN)의 기본 원리

**[7-1] 공간 구조 보존**
- 2차원 이미지를 그대로 2차원 배열로 표현
- 입력 픽셀을 패치(patch) 단위로 뉴런에 연결
- 전체 연결이 아닌 지역 연결(local connectivity) 사용

**[7-2] Local Connectivity(지역 연결성)**
- 하나의 출력 뉴런은 입력 이미지의 작은 패치에서만 정보를 받음
- 인접한 출력 뉴런들은 인접한 입력 픽셀 패치에서 파생됨
- 정보의 중복은 있지만 완전 연결은 아님

**[7-3] 공간 정보 보존 방식**
- 출력 레이어에서 가까운 뉴런 = 입력 레이어에서 가까운 픽셀에서 파생
- 이를 통해 공간 정보를 구조적으로 보존

---

### 8. Convolution Operation(합성곱 연산)

**[8-1] Convolution의 정의**
- 수학적 연산으로서의 합성곱
- 필터(filter)가 이미지 위를 단계별로 슬라이딩하며 연산 수행

**[8-2] Convolution 연산 과정**
1. 요소별 곱셈(Element-wise multiplication): 필터와 이미지 패치의 각 요소를 곱함
2. 덧셈: 모든 곱셈 결과를 더함
3. 편향 추가(Add bias): 편향 값을 더함
4. 비선형성 적용(Apply nonlinearity): 활성화 함수 적용
- Lecture 1과 동일한 연산이지만, 전체가 아닌 필터 수준에서만 수행

**[8-3] Sliding과 Scanning**
- 필터를 이미지 전체에 걸쳐 이동시킴 (예: 1픽셀 또는 2픽셀씩)
- 전체 이미지를 스캔하여 모든 패치를 커버

---

### 9. Feature Extraction with Convolution

**[9-1] 특징 추출의 메커니즘**
- 합성곱 연산이 어떻게 특징을 추출하는지가 핵심
- 필터가 이미지에서 찾고자 하는 패턴을 정의함

**[9-2] X 감지 예시**
- 문제 설정: 흑백 픽셀(+1 또는 -1)로 구성된 이미지에서 X를 분류
- 목표: 회전, 이동, 축소된 X도 감지 가능해야 함
- 해결 방법: 전역적 비교가 아닌 패치 단위 비교

**[9-3] X를 정의하는 특징들**
- 대각선(왼쪽→오른쪽): 첫 번째 특징
- 대각선(오른쪽→왼쪽): 두 번째 특징
- 교차점(crossing): 세 번째 특징
- 이 세 가지 특징을 패치 단위로 검출하면 X 식별 가능

**[9-4] 패치 단위 비교의 장점**
- 특징들이 정확히 같은 위치에 없어도 검출 가능
- 패치 수준에서 대략적인 매칭을 찾으면 전역 수준에서도 성공적 감지

---

### 10. Filter의 개념과 역할

**[10-1] Filter의 정의**
- 미니 이미지(mini image)
- 작은 2차원 숫자 배열
- 찾고자 하는 패턴을 표현하는 작은 이미지

**[10-2] Filter의 예시**
- 대각선 필터: 대각선 방향으로 +1, 나머지는 -1
- 교차 필터: 교차점 패턴을 표현
- 각 필터는 X를 구성하는 중요한 특성을 표현

**[10-3] Filter의 강건성(Robustness)**
- 새로운 X에 대해서도 특징 검출 가능
- 다양한 변형(회전, 이동 등)에 강건하도록 설계됨

---

### 11. Convolution의 수학적 표현

**[11-1] Convolution의 작동 방식**
- 공간 정보를 보존하며 이미지를 처리
- 큰 이미지를 작은 서브 이미지로 분해
- 서브 이미지 전체에서 핵심 정보 조각을 검색

**[11-2] 연산 과정**
- 찾고자 하는 것(필터)과 원본 이미지 간의 요소별 곱셈
- 이미지의 모든 지점에서 이 연산 수행

**[11-3] 3×3 필터 예시**
- 왼쪽→오른쪽 대각선 검출 필터
- 대각선은 +1, 나머지는 -1로 구성
- 이미지의 각 패치와 요소별 곱셈 수행
- 완벽한 매칭 시 모든 값이 1이 됨

**[11-4] 결과 계산**
- 요소별 곱셈 결과를 모두 더함
- 비선형성을 통과시킴 (Lecture 1과 동일)

---

### 12. Convolution 실행 예제

**[12-1] 5×5 이미지와 3×3 필터**
- 필터로 전체 이미지를 커버하기 위해 슬라이딩 필요
- 단계별로 이미지를 조각조각 스캔

**[12-2] 슬라이딩 프로세스**
1. 왼쪽 상단 모서리에 필터 배치
2. 3×3 패치와 요소별 곱셈 수행
3. 모든 항목을 더하여 결과 산출 (예: 4)
4. 출력 레이어의 해당 뉴런 위치에 결과 저장
5. 한 단계 이동하여 반복 (예: 다음 결과는 3)
6. 전체 이미지를 커버할 때까지 반복

**[12-3] 입력-필터-출력 관계**
- 입력 이미지 + 필터 → 출력 레이어 계산 가능
- 이 연산은 네트워크의 첫 번째 레이어를 정의함

---

### 13. Filter의 선택과 학습

**[13-1] Filter 선택의 중요성**
- 필터가 출력 품질을 결정하는 조정 가능한 요소
- 필터 선택이 전체 프로세스를 좌우함

**[13-2] Hand-engineered Filters**
- 오랫동안 사람들이 수동으로 필터를 설계함
- 학습 없이도 특정 목적을 달성하는 필터 설계 가능
- 예: 엣지(edge) 검출 필터 - 고휘도에서 저휘도로의 변화 감지 (미분 필터의 일종)

**[13-3] 다양한 Hand-engineered Filter 예시**
- Sharpening filter (선명화 필터): 중심을 증폭하고 가장자리 최소화
- Edge detection filter (엣지 검출 필터): 미분 형태, 선명화의 반대 효과
- Strong edge detection filter (강한 엣지 검출 필터)

**[13-4] Filter 가중치의 영향**
- 필터의 가중치를 바꾸면 다음 레이어의 출력이 완전히 변경됨
- 필터가 이미지에서 찾고자 하는 것을 정의하는 데 큰 영향

**[13-5] Zero의 의미**
- 필터에서 0은 해당 위치에서 정보를 찾지 않음을 의미
- 0 × 0 = 0 (널(null) 또는 수동적 정보)
- 해당 검출 위치에서 중요하지 않은 정보임을 나타냄

---

### 14. CNN의 구조와 학습

**[14-1] CNN의 목표**
- 이미지 데이터로부터 필터(특징)를 학습
- 왼쪽의 여러 이미지들로부터 오른쪽의 다른 클래스를 구별할 수 있는 필터 학습

**[14-2] CNN의 세 가지 주요 연산**
1. Convolutions (합성곱): 이미지 전체의 패치 단위 처리
2. Non-linearities (비선형성): 각 합성곱 후 적용
3. Pooling (풀링): 다운샘플링 연산

**[14-3] Pooling의 역할**
- 차원을 축소하여 수용 영역(receptive field) 확대
- 패치 단위로 작업할 때 깊이에 따라 패치 크기가 달라져야 함
- 이미지를 다운샘플링하면 같은 패치 크기로도 더 넓은 영역을 효과적으로 커버

**[14-4] 학습 프로세스**
- 이미지 세트로 모델 훈련
- 역전파(back propagation)를 통해 최적의 필터 가중치 학습
- 모든 연산을 정의하고 데이터를 보여주면 작업을 최적으로 해결하는 가중치 계산
- 최적화를 통해 이루어짐

---

### 15. CNN의 상세 구조

**[15-1] Hidden Layer의 뉴런 계산**
- 입력의 가중 합(weighted sum) 계산
- 필터와의 합성곱 적용
- 편향 추가
- 비선형성 적용

**[15-2] Local Connectivity의 중요성**
- 각 은닉층 뉴런은 입력의 패치만 볼 수 있음
- 이전 레이어의 패치에 있는 뉴런들만 입력으로 사용

**[15-3] 수학적 표현**
- 가중치 행렬(필터 또는 특징) 적용
- 예: 4×4 행렬 사용
- 요소별 곱셈 수행
- 출력 합산, 편향 추가, 비선형성 적용
- 요소별 곱셈과 덧셈 = 합성곱

---

### 16. Multiple Filters와 Volume

**[16-1] 여러 필터의 사용**
- 한 레이어는 하나의 필터만 사용하지 않음
- 전체 필터 세트를 사용하여 다양한 특징 검출
- 예: 한 레이어가 다양한 유형의 엣지를 찾을 수 있음

**[16-2] 출력 Volume**
- 하나의 2D 이미지가 아닌 전체 볼륨(volume) 출력
- 볼륨은 하나의 필터가 아닌 전체 필터 깊이에서 파생됨
- 이것이 CNN의 한 레이어를 구성함

**[16-3] 레이어 간 처리**
- 입력: 볼륨 (첫 레이어에서는 이미지)
- 처리: 해당 이미지에서 n개의 다른 필터 검출
- 출력: 이미지와 n개 필터의 합성곱 결과인 새로운 볼륨

---

### 17. Non-linearity (비선형성)

**[17-1] 비선형성의 필요성**
- 볼륨 생성 후 모든 요소를 비선형성을 통과시킴
- 표현력(expressivity)과 비선형 용량(non-linear capacity) 제공
- 처음 두 강의에서 다룬 핵심 개념

**[17-2] ReLU Activation**
- 이미지에 일반적으로 사용되는 비선형성: ReLU (Rectified Linear Unit)
- 작동 방식:
  - 음수 값을 모두 제거하고 0으로 압축
  - 양수 값은 그대로 유지
- 의미: 임계값 함수(thresholding function)의 일종
- 모든 것을 양수로 만드는 직관적인 의미

**[17-3] ReLU의 컴퓨터 비전적 해석**
- 컴퓨터 비전에서 임계값 함수로 해석 가능
- 모든 것을 없애고 양수로 압축하는 역할

**[17-4] Convolution vs Non-linearity 역할 구분**
- Convolution: 입력 내 특징 검출
- Non-linearity: 검출의 표현력을 증가시키고 비선형적으로 만듦
- 첫 단계에서는 더 이상 합성곱을 수행하지 않음
- 깊은 네트워크의 여러 레이어에서 이들을 순차적으로 깊이 방향으로 쌓음
- 각 레이어는 하나의 합성곱 연산을 수행하고 순차적으로 쌓임

---

### 18. Pooling (풀링)

**[18-1] Pooling의 목적**
- 차원을 다운스케일하는 것 이상의 의미는 없음
- 합성곱 후 출력 특징 세트의 크기를 줄임
- 특징을 다운스케일한 후 합성곱을 반복

**[18-2] Pooling의 효과**
- 다음 합성곱 시 더 큰 수용 영역(receptive field) 확보
- 입력 이미지가 작아지기 때문에 같은 필터 크기로도 더 넓은 영역 커버 가능

**[18-3] Max Pooling**
- 일반적인 기법
- 왼쪽 패치에서 최대값 선택
- 2×2 영역을 1×1 단일 최대값으로 축소
- 예: 2×2 → 1×1로 다운샘플링

**[18-4] Max Pooling의 문제점**
- 미분(derivatives)에 가혹함
- 4개 픽셀 중 3/4은 기울기가 전달되지 않음
- 최대값을 활성화하는 하나의 픽셀만 기울기 전달

**[18-5] 대안: Mean Pooling**
- 평균 풀링을 사용하면 전체 패치에 걸쳐 안정적인 기울기 유지
- 다른 형태의 다운풀링 방법도 고려 가능

---

### 19. Filter 시각화와 학습

**[19-1] 학습된 필터의 시각화**
- 모델 훈련 중 또는 후에 필터 시각화 가능
- CNN의 합성곱 레이어가 무엇을 검출하도록 최적화되는지 확인 가능

**[19-2] 얼굴 검출 CNN 예시**
- 첫 번째 레벨:
  - 기본적인 엣지 검출
  - 다양한 방향의 강도 변화 감지
- 중간 레벨:
  - 필터와 특징을 계층적으로 결합
  - 눈, 코, 귀와 같은 얼굴 특징이 나타나기 시작
- 상위 레벨:
  - 완전한 구조가 나타남
  - 전체 얼굴 형태 인식

**[19-3] 학습의 핵심**
- 필터를 직접 정의하지 않음
- 많은 데이터를 보여주고 얼굴 감지 정확도를 최대화하도록 요구
- 얼굴을 감지하지 못할 때만 패널티 부여
- 더 나은 얼굴 감지를 위해 필터를 계속 변경하도록 유도
- 모델이 스스로 최적의 필터 학습

---

### 20. CNN의 2단계 구조

**[20-1] Phase 1: Feature Extraction (특징 추출)**
- 학습된 필터를 사용하여 특징 추출
- 합성곱 레이어들로 구성

**[20-2] Phase 2: Classification/Detection (분류/감지)**
- Phase 1에서 학습한 특징을 사용
- 실제 감지 또는 분류 수행
- 이 경우 얼굴 감지 작업

---

### 21. Softmax Function

**[21-1] Softmax의 역할**
- N개의 출력을 합이 1이 되도록 압축
- 분류 작업에 필요한 확률 분포 생성

**[21-2] 확률 분포의 필요성**
- 분류를 위해서는 확률 분포가 필요
- 확률 분포 = 합이 1인 질량(mass)

---

### 22. CNN 코드 구현

**[22-1] Feature Extraction 부분**
- 첫 번째 레이어:
  - 32개의 feature maps (볼륨 깊이 32)
  - 32개의 필터가 학습됨
- 다운풀링: 공간 정보 축소
- 두 번째 합성곱 레이어:
  - 32개 특징을 받아 64개의 새로운 특징 학습
  - 더 작은 차원에서 수행

**[22-2] Classification 부분**
- Flatten: 모든 것을 평탄화하여 1차원 공간으로 전환
  - 공간 정보 학습이 완료되었으므로 가능
  - 1차원 예측을 원하기 때문
- Softmax:
  - 확률 분포 예측
  - 예: 10개 클래스 (숫자 0~9)

**[22-3] PyTorch 구현**
- 거의 동일한 코드 구조
- 합성곱 레이어 정의 (32개, 64개 특징)
- 평탄화 연산
- 최종 출력 차원으로 투영

### 23. 하이퍼파라미터 선택

**[23-1] 패치 크기 선택**
- 질문: 3×3 패치, 32개 feature maps, 3개 레이어를 선택한 이유?
- 답변: 과학보다는 예술에 가까움

**[23-2] 선택 가이드라인**
- 작업에 따라 특징 크기를 조정
  - 처음에는 작게 시작
  - 모델 깊이가 깊어질수록 공간 정보를 업샘플링하여 보존
- 이미지와 데이터셋에 따른 고려사항:
  - 인간으로서 어떤 스케일의 특징을 검출해야 하는지 이해
  - 시작과 끝의 해상도 정의
- 예시:
  - 2000×2000 픽셀 이미지에서 3×3은 너무 작을 수 있음
  - 작은 이미지에서는 3×3도 이미 상당한 해상도
- 매우 문제 의존적(problem-dependent)

### 24. 회전 불변성 문제

**[24-1] 질문**
- 정면 얼굴로 훈련한 분류기가 옆으로 회전한 얼굴도 잘 감지할 수 있는가?

**[24-2] 답변**
- 훈련 중 옆으로 회전한 얼굴을 보지 않았다면 잘 작동하지 않을 것
- 항상 정면 얼굴의 특징만 학습했기 때문
- 두 방향의 얼굴을 모두 보여줬다면 성공적으로 작동
- 두 경우 모두에서 특징을 추출하도록 학습했을 것

---

### 25. 변형 가능한 객체 처리

**[25-1] 질문**
- 거미줄이나 유체 시뮬레이션 같은 변형 가능한 객체에 어떻게 작동하는가?

**[25-2] 답변**
- 데이터로부터 학습 가능한지 확인해야 함
- 더 동적인 경우에도 적용 가능
- 강의 후반부에서 예시를 볼 수 있음

---

### 26. CNN의 다양한 응용

**[26-1] 기본 구조**
- 모든 CNN은 두 부분으로 분해 가능:
  1. Feature Extractor (특징 추출기): 왼쪽 부분
  2. Task-specific Module (작업별 모듈): 오른쪽 부분

**[26-2] 응용 가능성**
- 왼쪽(특징 추출기)을 고정하고 오른쪽만 변경하여 다양한 모델 생성 가능
- 동일한 특징 추출기 사용
- 오른쪽 부분만 바꿔서 매우 다른 작업 수행
- 실제로는 그렇게 다르지 않음

**[26-3] 다양한 응용 예시**
- Object detector (객체 검출기)
- Classifier (분류기)
- Segmentation model (세그멘테이션 모델)
- Probabilistic control model (확률적 제어 모델)
- Regressor model (회귀 모델) 등

---

### 27. Classification (분류)

**[27-1] 구조**
- 오른쪽 부분: 모든 특징에 대한 평탄화 연산
- n개의 다른 클래스로 분류

**[27-2] 예시: 의료 진단**
- 유방촬영 이미지에서 진단/비진단 이진 분류
- 몇 년 전 논문: CNN이 방사선 전문의를 능가할 수 있음을 입증

---

### 28. Object Detection (객체 감지)

**[28-1] Classification과의 차이**
- Classification:
  - 이진 또는 k개 클래스 문제
  - 예: "택시" 예측
- Object Detection:
  - 클래스 예측 + 위치 정보
  - 객체의 중심뿐 아니라 전체 바운딩 박스 예측
  - 장면의 모든 객체에 대해 수행

**[28-2] 출력의 유연성**
- 매우 유연한 출력 구조 필요
- 이미지에 택시가 하나만 있으면 하나의 바운딩 박스 출력
- 많은 객체가 있으면 여러 출력 생성
- 다양한 클래스, 위치, 개수, 순서 처리 가능

**[28-3] Object Detection의 복잡성**
- 박스는 이미지 어디에나 있을 수 있음
- 크기가 다양함
- 개수가 가변적 (0개부터 여러 개까지)

---

### 29. Naive Object Detection 방법

**[29-1] 첫 번째 시도**
- 이미지에 무작위 박스 하나 배치
- 해당 박스를 이전에 훈련한 CNN의 특징 추출기에 전달
- 분류 모델로 사용 (새로운 방법 사용 안 함)
- 반복적으로 무작위 박스 선택하여 분류

**[29-2] 문제점**
- 입력이 너무 많음 → 계산 비용이 매우 높음
- 실용적인 이미지에서 완전한 커버리지 불가능
- 해상도에 따라 박스 수가 기하급수적으로 증가

---

### 30. Heuristic-based Proposal

**[30-1] 개선 방법**
- 무작위 박스 대신 휴리스틱 방법 사용
- 이미지에서 "무언가 있어 보이는" 부분에 박스 그리기
- 이미지의 블롭(blob)을 찾는 간단한 휴리스틱

**[30-2] 장단점**
- 이미지에 대한 사전 지식 없이도 잠재적으로 흥미로운 영역 식별 가능
- 여전히 매우 느림: 많은 박스를 그려야 잘 작동
- 각 박스를 독립적으로 분류 모델에 입력해야 함

**[30-3] 근본적 문제**
- 두 부분의 문제를 완전히 분리함
  - 박스 제안(proposal) 모델
  - 분류 모델
- 이상적으로는 두 모델이 정보를 공유해야 함
- 좋은 박스를 예측하려면 무엇을 찾고 있는지 높은 수준에서 알아야 함

---

### 31. R-CNN (Regional CNN)

**[31-1] R-CNN의 개념**
- 같은 네트워크 내에서 영역(박스)을 학습
- 박스를 분류하는 모델과 동일한 네트워크에서 학습

**[31-2] R-CNN의 작동 방식**
- 원본 이미지에서 시작
- Region Proposal Network:
  - 이미지를 입력으로 받음
  - 영역 제안 학습
  - 이미지 내 중요한 영역의 특징 추출
- 이 영역들을 사용하여 박스 정의
- 동일한 분류기를 통과시킴 (영역 예측에 사용된 것과 동일한 특징 사용)

**[31-3] R-CNN의 장점**
- 전체 프로세스에서 좋은 정렬(alignment) - End-to-end
- 단일 순방향 패스(single forward pass)만 필요 → 매우 효율적
- 높은 정확도: 프로세스 전반에서 특징 공유

---

### 32. Semantic Segmentation (의미론적 세그멘테이션)

**[32-1] Segmentation의 정의**
- 이미지의 모든 픽셀에 대한 분류 예측
- 입력: RGB 이미지
- 출력: 각 픽셀의 분류 값을 가진 또 다른 이미지

**[32-2] Semantic Segmentation의 의미**
- "Semantic" = 의미론, 클래스 학습
- RGB 이미지의 왼쪽에서 각 픽셀의 의미(클래스) 학습

**[32-3] 예시**
- 왼쪽 소 이미지를 여러 카테고리로 분할:
  - 잔디 (grass)
  - 소 (cow)
  - 나무 (trees)
  - 하늘 (sky)

---

### 33. Segmentation Network 구조

**[33-1] 네트워크 구성**
- 왼쪽 절반 (First half):
  - 이전과 동일한 특징 추출기
  - 합성곱, 다운풀링, 비선형성으로 구성
- 오른쪽 절반 (Second half):
  - 1차원으로 다운샘플링하지 않음
  - 모든 것을 2차원으로 유지
  - 다운샘플링 대신 업샘플링 사용

**[33-2] 출력 학습**
- 또 다른 분류 문제
- 1차원으로 다운샘플링하지 않고 2차원 유지
- 업샘플링을 사용하여 원본 해상도로 복원

---

### 34. 자율주행 응용 사례

**[34-1] 문제 설정**
- 자율주행을 위한 신경망 학습
- 원시 인식 카메라 데이터 입력
- 노이즈가 있는 거리 뷰 맵 입력 (예: Google Maps의 조감도)

**[34-2] 인간 운전과의 유사성**
- 인간도 두 눈과 Google Maps만 사용하여 운전
- 새로운 도시에 가도 일정 수준의 내비게이션 정보만으로 운전 가능

**[34-3] 목표**
- 단일 조향각이 아닌 전체 확률 분포 추론
- 연속적인 분류 문제:
  - K개의 이산 클래스가 아님
  - 조향각은 연속 값 (예: -180° ~ +180°)

### 35. 자율주행 모델 구조

**[35-1] 학습 방법**
- 오늘 배운 것과 동일한 기법 사용
- 왼쪽 각 이미지를 동일한 유형의 모듈에 통과:
  - 합성곱 레이어
  - 비선형성
  - 다운샘플링
- 각 이미지의 출력에서 특징 추출

**[35-2] 제어 학습**
- 모든 특징을 1차원으로 결합
- 차량의 제어 매개변수 학습
- 역전파로 회귀(regression)
- 많은 주행 데이터를 보고 이 작업에 최적인 필터 학습

**[35-3] 결과**
- 이전에 주행한 적 없는 완전히 새로운 도로 주행 가능
- 맵 없이 새로운 도로 주행 가능

**[35-4] 기존 자율주행차와의 차이**
- Waymo, Google 자율차:
  - 자율주행 전에 인간이 먼저 도시를 주행해야 함
  - 사전 정의된 맵 필요
- 인간의 방식:
  - 새로운 도시에 처음 가도 운전 가능
  - 이전에 본 적 없어도 성공적 주행
- 이 모델도 인간과 유사한 행동 가능

---

### 36. CNN의 광범위한 영향

**[36-1] 다양한 응용 분야**
- 오늘 다룬 예시들을 넘어서는 광범위한 영향
- 모든 예시가 동일한 기본 기법에서 파생됨

**[36-2] 핵심 공통점**
- 연산 자체는 특징 학습 (필터 학습)
- 공간 해상도를 보존하거나 조절하기 위한 업샘플링/다운샘플링 조합

### 37. 강의 요약

**[37-1] Computer Vision의 기원**
- 컴퓨터 비전의 역사 고찰
- 이미지 표현 방법: 2차원 값 배열

**[37-2] 합성곱을 통한 학습**
- 2차원 배열에서 합성곱을 사용하여 의미 있는 데이터 추출
- 단일 합성곱 연산을 넘어 전체 네트워크로 구성

**[37-3] 다양한 응용**
- 합성곱 레이어 백본(backbone)을 사용한 여러 응용:
  - Classification (분류)
  - Detection (감지)
  - Segmentation (세그멘테이션)
  - 기타 다양한 작업
